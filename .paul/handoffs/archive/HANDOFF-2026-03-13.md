## Bugs in 04-01 (must fix before re-presenting checkpoint)

### Bug 1: Wrong coordinate space — all cards replicate first card's glow
**Symptom:** Only Pratt & Whitney (first card) tracks correctly. All other cards show the same gradient position as the first.

**Root cause:** `--mouse-x` and `--mouse-y` are set on `<main>` (panel-relative coords), but each card's `::before` background gradient renders `at var(--mouse-x) var(--mouse-y)` relative to *that card's* own origin. So all cards use the same coords but interpret them relative to themselves — only the first card (near panel top-left) approximately aligns.

**Fix:** In `page.tsx` `onMouseMove`, instead of setting vars on `<main>`, use `querySelectorAll('.flashlight-card')` to set vars on each card individually, computing coords relative to each card:
```tsx
onMouseMove={(e) => {
  const cards = e.currentTarget.querySelectorAll('.flashlight-card')
  cards.forEach((card) => {
    const rect = (card as HTMLElement).getBoundingClientRect()
    ;(card as HTMLElement).style.setProperty('--mouse-x', `${e.clientX - rect.left}px`)
    ;(card as HTMLElement).style.setProperty('--mouse-y', `${e.clientY - rect.top}px`)
  })
}}
```

### Bug 2: Full card area glows — should only be the border
**Symptom:** The entire card shows the radial gradient, not just the border edge.

**Root cause:** Experience card root div has no background color class. The `::before` (z-index: 0) is behind the card, but with no opaque background on the card itself, the gradient bleeds through the whole card area. Project cards are fine because they have `bg-bg-raised`.

**Fix:** Add `bg-bg` to the experience card root div in `experience.tsx`:
```tsx
className={`flex flex-col flashlight-card bg-bg rounded-xl p-5 cursor-pointer ${className ?? ''}`}
```
`bg-bg` is `#080808` — same as the panel background — so the card interior is covered and only the 1px border edge of the `::before` is visible.
