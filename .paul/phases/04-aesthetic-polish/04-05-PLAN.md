<objective>
## Goal
Change the CRT panning scan line from a faint dimming overlay to a color-inverting line using `mix-blend-mode: difference`.

## Purpose
A color-inverting scan line is more visually striking and authentic — as the line passes, a brief "negative" band appears, like a real CRT phosphor sweep. The current `rgba(255,255,255,0.04)` dim is nearly imperceptible; inversion makes the effect clearly readable at any content density.

## Output
- Modified: `src/app/globals.css` — `.crt-scanline-moving` background changed to `white`, `mix-blend-mode: difference` added
</objective>

  <action>
    In `.crt-scanline-moving`:
    - Change `background: rgba(255, 255, 255, 0.04)` to `background: white`
    - Add `mix-blend-mode: difference`

    `mix-blend-mode: difference` with a white source computes `|255 - dst|` per channel,
    which is exact color inversion of whatever is rendered underneath.

    Keep ALL other properties unchanged: height (3px), z-index (11), animation reference,
    will-change, and position (inherits `position: absolute; inset: 0` from `.crt-overlay > div`).

    Do NOT change any other `.crt-*` rules, keyframes, or any other file.
