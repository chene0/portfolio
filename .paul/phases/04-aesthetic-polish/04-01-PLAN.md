<objective>
## Goal
Add a cursor flashlight effect: card borders are invisible by default and revealed near the cursor via a radial gradient that tracks mouse position.

## Purpose
The current card borders are always visible at low opacity. The flashlight makes the portfolio feel crafted and alive — the site responds to the visitor's presence. Cards emerge from the dark as the cursor approaches, rewarding exploration.

## Output
- `globals.css`: `.flashlight-card` CSS class using `::before` pseudo-element + CSS custom properties
- `page.tsx`: mouse tracking on desktop panel, sets `--mouse-x`/`--mouse-y` CSS vars
- `experience.tsx`: swap static border for flashlight-card class
- `projects.tsx`: swap static border for flashlight-card class
</objective>

<task type="auto">
  <name>Task 1: Add flashlight CSS class and mouse tracking</name>
  <files>src/app/globals.css, src/app/page.tsx</files>
  <action>
    **globals.css — add `.flashlight-card` class:**

    ```css
    .flashlight-card {
      position: relative;
      border-radius: 12px; /* matches rounded-xl */
    }

    .flashlight-card::before {
      content: '';
      position: absolute;
      inset: -1px;
      border-radius: inherit;
      background: radial-gradient(
        500px circle at var(--mouse-x) var(--mouse-y),
        rgba(255, 255, 255, 0.08),
        transparent 50%
      );
      z-index: 0;
      pointer-events: none;
    }

    /* Ensure card content stays above ::before */
    .flashlight-card > * {
      position: relative;
      z-index: 1;
    }
    ```

    Add this after the existing `.img-shimmer` block.

    **page.tsx — mouse tracking on desktop main:**

    In the `Home` component, add a `useCallback` ref handler or inline `onMouseMove` on the desktop `<main>` element:

    ```tsx
    const handleMouseMove = (e: React.MouseEvent<HTMLElement>) => {
      const rect = e.currentTarget.getBoundingClientRect()
      e.currentTarget.style.setProperty('--mouse-x', `${e.clientX - rect.left}px`)
      e.currentTarget.style.setProperty('--mouse-y', `${e.clientY - rect.top}px`)
    }
    ```

    Add `onMouseMove={handleMouseMove}` to the desktop `<main>` element only (the one with `hidden md:flex`). Do NOT add to mobile layout — mobile has no cursor.

    The CSS vars cascade down to all `.flashlight-card` descendants, which use them for the gradient coordinates.
  </action>
  <verify>
    Run `npm run build` — no errors.
  </verify>
  <done>Foundation for AC-1 and AC-2 — CSS class and mouse tracking in place</done>
</task>
