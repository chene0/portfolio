<objective>
## Goal
Add restrained, tasteful motion to the portfolio: panel tab transitions, staggered list entrances, and hover micro-interactions — all using Framer Motion.

## Purpose
The site is visually solid after 03-01 and 03-02 but still static. This plan adds the "alive" quality — smooth panel transitions when switching tabs, content that enters gracefully, and subtle hover feedback. The vibe is restrained and refined, not showy: 200–300ms easing, no bounce, no parallax.

## Output
- `page.tsx`: AnimatePresence tab transitions, staggered entrance for skills
- `projects.tsx`: staggered card entrance + hover lift
- `awards.tsx`: staggered list entrance
</objective>

## AC-1: Tab Panel Transitions
```gherkin
Given the user is on any tab
When they click a different tab
Then the current panel fades/slides out and the new panel fades/slides in
And the transition takes ~220ms with ease-out
```

## AC-2: Panel Content Entrance
```gherkin
Given any tab panel is activated (first load or tab switch)
When the panel content mounts
Then the content fades up from y+8 to y+0 over ~220ms
```

## AC-3: Project Card Hover
```gherkin
Given the Projects tab is active
When the user hovers a project card
Then the card lifts slightly (translateY -2px) with a subtle shadow increase over 150ms
```

## AC-4: Awards List Entrance
```gherkin
Given the Awards section is visible in Work tab
When the Work tab mounts
Then award items stagger in with 50ms delay between each item
```

## AC-5: No Layout Shifts
```gherkin
Given any animation is running
When the animation plays
Then no layout shifts occur — content does not push other elements around during animation
```

  <name>Task 1: Tab panel transitions with AnimatePresence</name>
  <files>src/app/page.tsx</files>
  <action>
    Wrap `PanelContent` renders in `AnimatePresence mode="wait"` so panels transition out before the next mounts.

    **In PanelContent:**
    - Import `motion, AnimatePresence` from framer-motion
    - Wrap the entire return of PanelContent in `<AnimatePresence mode="wait">`
    - Wrap each panel (AboutPanel, WorkPanel, ProjectsPanel) render in a `<motion.div>` with `key={activeTab}`:
      ```tsx
      <motion.div
        key={activeTab}
        initial={{ opacity: 0, y: 8 }}
        animate={{ opacity: 1, y: 0 }}
        exit={{ opacity: 0, y: -8 }}
        transition={{ duration: 0.22, ease: 'easeOut' }}
      >
        {/* panel content */}
      </motion.div>
      ```

    **Note:** PanelContent is called in both desktop main and mobile main — make sure AnimatePresence wraps correctly in both call sites, or apply it inside PanelContent itself (preferred — one change, both layouts benefit).

    Do NOT: add bounce, spring, or any easing that feels "playful". Keep it subtle — this is a professional portfolio.
  </action>
  <verify>
    `npm run build` passes. Switch tabs in browser — panels transition with a clean fade+slide.
  </verify>
