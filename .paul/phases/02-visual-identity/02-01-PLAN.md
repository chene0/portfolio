<objective>
## Goal
Remove DaisyUI entirely and establish a clean design system — pure Tailwind + CSS custom properties. Set the "Liminal Late Night" palette, typography, and design tokens that all subsequent Phase 2 plans build on.

## Purpose
DaisyUI's opinionated theming, markup, and utility classes conflict with the fully custom visual direction. Removing it now — before any component rebuilds — means Phase 2 starts from a clean foundation with complete design freedom. The site will look visually broken after this plan (DaisyUI classes lose their styles), which is expected: Plans 02-02 through 02-04 replace every component.

## Output
- DaisyUI removed from package.json, tailwind.config.ts
- `tailwind.config.ts` extended with liminal color palette + font families
- `globals.css` rewritten: CSS custom properties, base styles, font smoothing
- `layout.tsx`: Space Grotesk + Inter fonts, `data-theme` removed, theme toggle removed, metadata updated
- Site renders dark background with correct typography — components are unstyled but functional
</objective>

<task type="auto">
  <name>Task 3: Update Layout — Fonts, Remove Toggle and DaisyUI Refs</name>
  <files>src/app/layout.tsx</files>
  <action>
    Rewrite layout.tsx:

    ```tsx
    import type { Metadata } from "next";
    import { Inter, Space_Grotesk } from "next/font/google";
    import "./globals.css";

    const inter = Inter({
      subsets: ["latin"],
      variable: "--font-inter",
    });

    const spaceGrotesk = Space_Grotesk({
      subsets: ["latin"],
      variable: "--font-space-grotesk",
      weight: ["300", "400", "500", "600"],
    });

    export const metadata: Metadata = {
      title: "Ethan Chen",
      description: "Computer Engineering student at UWaterloo. Building software.",
    };

    export default function RootLayout({
      children,
    }: Readonly<{
      children: React.ReactNode;
    }>) {
      return (
        <html lang="en">
          <body className={`${inter.variable} ${spaceGrotesk.variable}`}>
            {children}
          </body>
        </html>
      );
    }
    ```

    Note: `min-w-full`, `max-w-full`, `bg-base-100`, `text-base-content` removed —
    these were DaisyUI classes. Base styles are now handled by globals.css.
  </action>
